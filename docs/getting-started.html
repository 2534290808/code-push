Getting Started

CodePush is a cloud service that enables Cordova and React Native developers to deploy mobile app updates directly to their users’ devices. It works by acting as a central repository that developers can publish certain updates to (e.g. JS, HTML, CSS and image changes), and that apps can query for updates from (using our provided client SDKs). This allows you to have a more deterministic and direct engagement model with your end-users, while addressing bugs and/or adding small features that don’t require you to re-build a binary and/or re-distribute it through any public app stores.

1. Request beta acces

While the CodePush service is in beta, we are running a “closed-beta” that requires developers to request access to the service. Simply submit your request, using either a GitHub or Microsoft Account, and we’ll notify you as soon as you’ve been accepted.

2. Install the CodePush CLI

Once you have an account, you’ll need the CodePush CLI in order to begin managing your apps and update. The CLI is built using NodeJS, and can be easily acquired using the following in your command prompt or terminal: npm install -g code-push-cli

Note: On OSX and Linux, you may need to prefix this command with sudo

3. Log in via the CLI

All management activities require authentication, and therefore, you need to login with the account you requested access for. To do this, simply type the following via the CLI, type code-push login. This will walk you through logging in with your GitHub or Microsoft Account.

4. Register your app with the service

In order to let the service know about your app, simply register it using a recognizeable name. For example: code-push app add MyApp

5. CodePush-ify your app

Add the appropriate CodePush clients SDKs to your app, and configure them to query for updates against the app deployment created above. The following provide details on how to do this for each unique app type:

  • Cordova
  • React Native (iOS)

6. Release an app update

After making changes to your app’s code or assets, push the update to your staging environment via the following command: code-push release <appName> <package> <appStoreVersion> to push those changes to the service.

7. Run your app

And thats it! All users running your app will receive the update using the experience you configured in step #4. For more details, refer to the CLI and client SDK documentation (Cordova, React Native).
